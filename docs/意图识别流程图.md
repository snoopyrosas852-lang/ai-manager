# 意图识别层 - 运作流程图（OpenClaw 风格重构后）

> 重构说明：意图由 Skill 的 description + instructions 隐式定义，LLM 根据注入的 Skill 列表一次推理完成工具选择。

## 一、整体流程

```mermaid
flowchart TB
    subgraph 用户侧
        A[用户输入消息] --> B{是否有对话历史?}
    end

    subgraph 意图识别层
        B -->|有| C[合并: 历史 + 当前消息]
        B -->|无| D[仅当前消息]
        C --> E[意图切换检测]
        D --> E

        E --> F{触发切换规则?}
        F -->|是| G[重置上下文]
        F -->|否| H[保留上下文]
        G --> I[意图识别]
        H --> I

        I --> J[调用 LLM + 意图体系 + Prompt]
        J --> K{识别结果}

        K -->|置信度低/信息不足| L[多轮对话规则]
        L --> M[生成追问]
        M --> N[返回追问给用户]
        N --> A

        K -->|置信度高| O[输出: skill, intent, apiParams]
        O --> P[路由到对应 Skill]
        P --> Q[调用工具/API]
        Q --> R[生成回复]
        R --> S[返回给用户]
    end

    subgraph 配置层_后台维护
        T[Skill description+instructions] -.-> J
        U[系统 Prompt] -.-> J
        V[多轮对话规则] -.-> L
        W[意图切换规则] -.-> E
    end
```

---

## 二、各环节说明

| 环节 | 输入 | 输出 | 配置来源 |
|------|------|------|----------|
| **意图切换检测** | 当前消息 + 历史 | 是否切换、是否重置上下文 | 意图切换规则（触发关键词） |
| **意图识别** | 消息（含历史） | skill、intent、apiParams、confidence | 意图体系、Prompt 配置 |
| **多轮对话** | 识别结果（低置信度） | 追问文案 | 多轮对话规则 |
| **Skill 路由** | skill ID | 具体工具调用 | 意图体系（intent → skillIds） |

---

## 三、配置与运行时的关系

```mermaid
flowchart LR
    subgraph 后台配置_开发时
        A1[意图体系] --> A2[意图定义]
        A2 --> A3[名称/描述/示例/Skill]
        A4[Prompt 配置] --> A5[意图识别 Prompt]
        A4 --> A6[意图切换 Prompt]
        A4 --> A7[多轮澄清 Prompt]
        A8[多轮规则] --> A9[条件 → 追问]
        A10[切换规则] --> A11[关键词 → 动作]
    end

    subgraph 运行时
        B1[用户消息] --> B2[意图识别服务]
        B2 --> B3[读取配置]
        B3 --> A1
        B3 --> A4
        B3 --> A8
        B3 --> A10
        B2 --> B4[LLM 推理]
        B4 --> B5[结构化输出]
    end
```

## 四、意图定义如何参与识别

```mermaid
flowchart TB
    subgraph 意图体系_你维护的
        I1[意图A: 订单查询]
        I2[意图B: 商品查询]
        I3[意图C: 退货查询]
        I1 --> I1a[skillIds: order_query]
        I2 --> I2a[skillIds: product_query]
        I3 --> I3a[skillIds: return_query]
    end

    subgraph 识别过程
        M[用户: 查一下我的订单] --> P[Prompt 模板]
        P --> |注入| I1
        P --> |注入| I2
        P --> |注入| I3
        P --> LLM[大模型]
        LLM --> OUT["输出: { skill: order_query, intent: 订单查询 }"]
    end

    OUT --> ROUTE[路由到 order_query 技能]
```

**要点**：意图体系中的「名称、描述、示例」会被拼进 Prompt，供大模型参考，从而输出对应的 skill 和 intent。

## 五、多轮对话与意图切换

```mermaid
flowchart TB
    subgraph 多轮对话
        U1[用户: 查一下] --> R1[信息不足]
        R1 --> C1[匹配多轮规则]
        C1 --> Q1[追问: 您想查订单、商品还是其他?]
        Q1 --> U2[用户: 订单]
        U2 --> R2[意图识别: order_query]
    end

    subgraph 意图切换
        U3[用户: 查订单] --> R3[当前: order_query]
        U4[用户: 换个，查商品] --> C2[匹配切换规则]
        C2 --> R4[重置上下文]
        R4 --> R5[意图识别: product_query]
    end
```
